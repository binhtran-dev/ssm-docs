## Items to fix

- [ ] **PracticeContract.java** — `queryParams` `@Parameter` is declared as `ParameterIn.PATH` but should be `ParameterIn.QUERY` (or `hidden = true`). The method uses `@RequestParam Map<String, String>` with `@GetMapping("")` (no path variable). Pre-existing issue, not introduced by DSSC-8643. Fix: change to `hidden = true` since SpringDoc auto-discovers `@RequestParam Map` parameters.
- [ ] **PracticeControllerIT.java** — `testGetAllPracticeWithSurgeryReadScope` only asserts the response is not 401/403, so it will still pass on 5xx/404 and won't reliably validate the expanded `@PreAuthorize`. Fix: assert `HttpStatus.OK` directly (e.g. `assertThat(restTemplate.getStatusCode()).isEqualTo(HttpStatus.OK)`) and remove the unused `assertNotEquals` import.
- [ ] **PracticeControllerIT.java** — `testGetAllPracticeFilterByPrimaryMinistry` only checks the result is non-empty, so it would pass even if the API ignores `primaryMinistry` entirely. Fix: insert a second Practice with a different `primaryMinistry` (e.g. `MOKCM`), then assert the returned list size is 1 and that the returned item has `primaryMinistry` equal to `TXAUS`.
- [ ] **PracticeControllerIT.java / Repository projection** — `testGetPracticeReturnsFaxNumber` expects `faxNumber` from `GET /practices/{id}`, but the repository aggregation projection used by `findPracticeById`/`findPracticeByEuid` only projects `id, name, address, suite, city, state, zipCode, euid, primaryMinistry, primaryHospital, distributionEmail`. `faxNumber` is not included, so it will always be `null` and the test will fail at runtime. Fix: add `faxNumber` to the aggregation projection in the repository, or switch the endpoint to fetch the full `Practice` document and map it.
